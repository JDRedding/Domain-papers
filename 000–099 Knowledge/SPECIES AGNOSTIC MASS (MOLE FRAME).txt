📘 Style Guide: Operators vs Functions
-------------------------------------
This is the rulebook for when to use brackets vs parentheses,
 and how to keep notation consistent across systems.

1. Use brackets for operators

Operators act on objects.
Brackets denote operator domain.

	Ξ[…]
	Ξᵣ[…]
	Γ[…] (later, for continuum limit)
	D[…] (if you define a differential operator)

Example:

	Ξ[mole_[unit]^X]

2. Use parentheses for functions

Functions take arguments.
Parentheses denote function inputs.

	r(…)
	f(…)
	g(…)

Example:

	r(m_μ^X, M_[unit])

3. Use subscripts for operator modes

Operator modes are semantic tags.

	Ξᵣ — relational
	Ξ_s — structural
	Ξ_d — dynamic

This keeps operator family clean and extensible.

4. Use superscripts for species anchors

Species anchors belong in superscripts.

	m_μ^X
	mole_[unit]^X

This keeps micro‑domain indexing consistent.

5. Use bracketed subscripts for macro‑units

Macro‑units belong in bracketed subscripts.

	M_[unit]
	mole_[unit]^X

This visually separates “unit choice” from “species choice.”
